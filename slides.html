Le problème

Une facture sur trois contient des mentions manquantes

📄

13 mentions obligatoires

L'Art. L441-9 du Code de Commerce impose 13 éléments sur chaque facture. La plupart des PME ne les connaissent pas toutes.

⚖️

Sanctions réelles

Amende jusqu'à 75 000€ pour les personnes morales. Une mention SIRET ou TVA manquante suffit.

🕐

Vérification manuelle

Un expert-comptable vérifie ça à la main, facture par facture. Lent, coûteux, pas scalable.

💡

L'opportunité

Un modèle de langage lit une facture en 10 secondes et identifie chaque mention mieux qu'un humain.

Architecture IA

Un agent, pas un simple appel LLM

AVANT — LLM wrapper
user: "Analyse cette facture,
      réponds en JSON"

claude: {
  "missing": [...],
  "score": 65
}
MAINTENANT — Agent tool use
claude → flag_mention("SIRET", false)
claude → flag_mention("TVA", true, "FR123...")
claude → flag_mention("Date", true, "09/11/25")
       ... × 13 mentions
claude → finalize(score=62, status="attention")
TOUR 0
Claude reçoit la facture + les tool definitions. Il appelle flag_mention() 13 fois en parallèle.
→
COLLECTE
L'Edge Function reçoit chaque tool call, compile les listes present[] et missing[].
→
FINALIZE
Claude appelle finalize() avec le score global, le statut et l'email de relance.
Tool Use

Claude décide quels tools appeler et dans quel ordre

flag_mention(mention, present, detail)

Appelé pour chacune des 13 mentions obligatoires. Claude passe true/false et la valeur trouvée ou l'explication de l'absence.

finalize(score, status, summary, relance_email)

Appelé une seule fois après toutes les vérifications. Produit le score 0-100, le statut et l'email de relance si nécessaire.

Résultat observé

15 tool calls en 1 tour, ~10 secondes. Claude batche toutes les vérifications en parallèle sans orchestration explicite.

Extensibilité

Ajouter un nouveau tool lookup_legal_article() permettrait à Claude de citer la jurisprudence exacte pour chaque mention manquante.

Runs capturés
Langsmith trace chaque run avec : type d'input (image/texte), longueur du contenu, score de conformité, nombre de mentions manquantes, latence totale et erreurs éventuelles. Le trade-off latence +6s est visible et mesurable — c'est le coût du pattern agent vs un appel direct.
